# 2026-04-19

## 微信登录/注册按钮修复

**问题**：login 和 register 页面的微信按钮用了 `<button>` + `!important` flex 布局，小程序不支持，导致图标文字竖排乱掉。

**修复方案**：
- 改用 `<view bindtap>` 替代 `<button open-type>`
- 图标改为纯 CSS 圆形（`wechat-icon-circle`）+ 文字「微」，不依赖 SVG
- JS 改为 `wx.login()` 获取 code（替代原来的 `getPhoneNumber` 回调）
- 去掉所有 `!important`，WXSS flex 直接生效

**文件改动**：
- `pages/login/login.wxml` - 微信按钮结构
- `pages/login/login.wxss` - 微信按钮样式（新增 wechat-icon-circle）
- `pages/login/login.js` - onWechatLogin 改用 wx.login
- `pages/register/register.wxml` - 同上
- `pages/register/register.wxss` - 同上
- `pages/register/register.js` - onWechatRegister 改用 wx.login

## 开屏页强制从头开始

**需求**：每次刷新都从开屏动画从头开始，点击箭头后必须登录才能进入主界面。

**修复方案**：
- 移除 `index.js` onLoad 的自动登录检查，每次都播动画
- `onStart` 直接跳登录页，不判断 token
- 移除 `main.js` onLoad 的登录检查

**文件改动**：
- `pages/index/index.js` - 移除自动跳转逻辑
- `pages/main/main.js` - 移除登录检查

## 登录注册页优化

**需求**：
1. 去掉微信登录/注册按钮
2. 输入框下划线缩短，两边空白对齐
3. 后端验证——没注册的账号无法登录
4. 整体布局居中，左右留白一致
5. 标签与输入内容间距合理
6. 注册完跳转登录页，必须通过登录页进入

**修复方案**：
- 移除 login.wxml 和 register.wxml 的微信按钮和分割线
- 容器统一 `padding: 60rpx 80rpx`，使用 `box-sizing: border-box`
- 输入框和按钮宽度 100%，内部元素不再额外加 margin
- 使用 `registeredUsers` 本地存储模拟后端用户表
- 注册时检查账号是否已存在
- 登录时验证账号是否存在、密码是否正确
- 标签固定定位，`top: 8rpx`，输入框 `padding-top: 56rpx`，间距合理
- 注册成功后提示并跳转登录页，不直接进主界面

**文件改动**：
- `pages/login/login.wxml` - 移除微信按钮，固定标签类
- `pages/login/login.wxss` - 重写布局，调整标签和输入框间距
- `pages/login/login.js` - 添加账号存在性验证
- `pages/register/register.wxml` - 移除微信按钮，固定标签类
- `pages/register/register.wxss` - 重写布局，调整间距
- `pages/register/register.js` - 添加账号重复检查，注册后跳转登录页
